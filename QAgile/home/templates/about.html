RBS QA CoE Initiative
Goal:

To build an integrated system for QA organization that integrates processes, tools and people and act as one stop shop for QA and RBS.
Current Features:

Manage Team - This section is ideally for admin to manage team data that is being used in RFS creation. We can manage people, domains, roles, locations and rates. All this data is input to RFS.

Create RFS - There is an RFS creation and workflow in place. Anyone (preferably project team) starts QA engagement by creating RFS from 'Create Request for Service'. RFS can be saved, submitted, approved, rejected. It implements almost all RFS features that we do manually.

Please send email to report any bug, feature suggestions or comments
Future Features:

Tenrox Integration - Integrate tenrox data for automated effort and cost analysis, planned vs actuals etc.

Jira/ALM Integration: To bring in uniform analytics and reporting across projects.

SharePoint Integration - Keep documents stored in SharePoint behind the scene and manage documents from single point.

QA Project Execution Workflows: Start to finish project and process execution.
